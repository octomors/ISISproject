# ТЗ: веб-платформа коллективной оптимизации кода

## 1) Назначение системы
Создать веб-платформу, где публикаторы размещают задачи по оптимизации кода, а оптимизаторы отправляют решения (патчи/архивы), которые автоматически проверяются и ранжируются в лидерборде.

Ниже обновление ТЗ (ключевые изменения):

## 2. Роли и права

### 2.1 User (единая роль)
Один и тот же пользователь может одновременно:

- публиковать задачи,
- отправлять решения в чужие задачи,
- голосовать за чужие решения,
- получать и тратить баллы.

### 2.2 Admin
- настраивает параметры платформы (стоимость публикации, награда, лимиты),
- модерирует контент и спорные случаи,
- при необходимости завершает/отменяет задачи вручную.

## 3. Экономика баллов
У пользователя есть баланс баллов (points_balance).

Публикация задачи платная: при создании задачи списывается фиксированная стоимость публикации (publish_cost).

Награду назначает платформа (platform_reward), а не автор задачи.

После завершения задачи победителю начисляется награда по правилам платформы.

Если баллов недостаточно — публикация запрещена.

## 5. Бизнес-правила
Пользователь не может голосовать за собственное решение.

Голосовать можно только в активной задаче и до дедлайна.

Победитель — автор решения с максимальным количеством голосов на момент дедлайна.

При равенстве голосов выше тот, чье решение раньше получило итоговый результат (tie-break по времени).

После завершения задачи голосование и отправка новых решений запрещены.

## 6. Сценарии пользователя

### 6.1 Публикация задачи
User вводит:

- ссылку на репозиторий,
- дедлайн,
- описание.

Система проверяет баланс.

При успешной проверке списывает publish_cost.

Создает задачу со статусом active.

### 6.2 Участие в задаче
User выбирает активную задачу.

Отправляет свое решение (Submission).

### 6.3 Голосование
User просматривает чужие решения в задаче.

Выбирает одно решение и отдает голос.

Система валидирует ограничения и сохраняет голос.

### 6.4 Завершение задачи
По дедлайну (или вручную админом) задача закрывается.

Система фиксирует победителя по голосам.

Система начисляет platform_reward победителю.

Обновляется рейтинг/лидерборд.
